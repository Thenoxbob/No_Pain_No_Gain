Matériel Réseau

Chaque équipement joue un rôle précis. Comprendre leur fonctionnement permet de concevoir et dépanner n'importe quelle infrastructure.

01 Switch (Commutateur)

Le switch est l'équipement central d'un réseau local. Il opère en couche 2 du modèle OSI et connecte les machines d'un même réseau en utilisant les adresses MAC pour diriger les trames uniquement vers le port destinataire.

Contrairement au hub (qui diffusait à tous les ports créant des collisions), le switch maintient une table MAC : à chaque port correspond une adresse MAC apprise dynamiquement. Le trafic est isolé et les performances sont maximisées.

# Commandes de base sur un switch Cisco (IOS)
Switch> enable
Switch# show mac address-table
Vlan    Mac Address        Type    Ports

   1    b8:27:eb:4d:2e:1a  DYNAMIC Gi0/3
  10    00:1a:2b:3c:4d:5e  DYNAMIC Gi0/5

02 Routeur

Le routeur opère en couche 3 du modèle OSI. Son rôle est d'interconnecter des réseaux IP différents et de choisir le meilleur chemin pour acheminer chaque paquet à destination.

Il maintient une table de routage — soit configurée statiquement, soit construite dynamiquement via des protocoles de routage (OSPF, BGP, RIP). Il effectue également la translation d'adresses (NAT) permettant à un réseau privé d'accéder à Internet via une seule IP publique.

# Afficher et comprendre une table de routage Linux
$ ip route show

default via 192.168.1.1 dev eth0          # tout trafic non connu → passerelle
192.168.1.0/24 dev eth0 proto kernel      # réseau local direct
10.0.0.0/8 via 192.168.1.254 dev eth0    # route statique vers réseau privé

# Sur un routeur Cisco IOS
Router# show ip route
C    192.168.1.0/24  directly connected, Gi0/0
O    10.10.10.0/24   [110/2] via 192.168.1.254  # route OSPF
S*   0.0.0.0/0       [1/0] via 203.0.113.1       # route par défaut statique

03 Firewall (Pare-feu)

Le firewall filtre le trafic réseau selon des règles de sécurité (politiques). Il peut être matériel (appliance dédiée) ou logiciel. C'est le premier rempart de toute infrastructure réseau.

Les NGFW (Next-Generation Firewall) vont bien au-delà du filtrage par ports : inspection profonde des paquets (DPI), identification des applications (peu importe le port), prévention d'intrusion (IPS) intégrée, déchiffrement SSL, filtrage par utilisateur.

# Règles iptables Linux — exemples courants

# Autoriser le trafic SSH entrant (port 22)
$ iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# Autoriser le trafic établi et lié (connexions déjà ouvertes)
$ iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# Bloquer tout le reste en entrée (règle par défaut)
$ iptables -P INPUT DROP

# Lister les règles actives
$ iptables -L -v -n

04 Point d'accès Wi-Fi (AP)

05 Serveur

Un serveur est une machine (ou une machine virtuelle) dédiée à fournir des services aux clients du réseau. Il se distingue par sa haute disponibilité, ses composants redondants (PSU, disques en RAID, interfaces réseau en bonding) et ses ressources dimensionnées pour la charge.

TypeRôleLogiciels courants
Serveur WebHéberger des sites et applicationsApache, Nginx, IIS
Serveur DNSRésolution de noms de domaineBIND9, Unbound, Windows DNS
Serveur DHCPDistribution automatique d'adresses IPISC DHCP, dnsmasq, Windows DHCP
Serveur de fichiersPartage de fichiers en réseau (SMB, NFS)Samba, NFS, Windows File Server
Serveur mailEnvoi, réception et stockage d'e-mailsPostfix, Dovecot, Exchange, Zimbra
Serveur d'annuaireGestion centralisée des comptes et droitsActive Directory (AD), OpenLDAP
Serveur de supervisionSurveillance de l'infrastructure réseau et systèmeZabbix, Nagios, Prometheus, PRTG
Serveur de virtualisationHébergement de VMs ou conteneursVMware ESXi, Proxmox, Hyper-V
// haute disponibilité
En production, on évite les SPOF (Single Point Of Failure). Un serveur critique est généralement doublé (actif/passif ou actif/actif), avec un load balancer devant qui répartit la charge et bascule automatiquement en cas de panne.

06 Câblage & Médias de transmission

Le câblage physique est souvent sous-estimé. Un câble mal sertis ou de catégorie inadaptée peut limiter les performances de toute une infrastructure, peu importe la qualité des équipements actifs.

En entreprise, on applique la norme TIA-568 (Amérique) ou ISO/IEC 11801 (Europe) pour le câblage structuré. Chaque poste est relié à un panneau de brassage (patch panel) dans une baie 19", lui-même connecté aux switches actifs. Cette organisation facilite les interventions et garantit la traçabilité.

07 Carte réseau (NIC)

La NIC (Network Interface Card) est l'interface physique qui connecte un équipement au réseau. Elle possède une adresse MAC unique (48 bits, gravée en usine) qui sert à l'identification sur le réseau local.

Les serveurs utilisent souvent plusieurs NIC en bonding/teaming pour cumuler la bande passante ou assurer la redondance en cas de panne d'une carte ou d'un câble.

# Afficher les interfaces réseau et leurs adresses MAC
$ ip link show
2: eth0: <BROADCAST,MULTICAST,UP> mtu 1500
    link/ether a4:c3:f0:1b:2d:3e brd ff:ff:ff:ff:ff:ff

# Configurer un bonding (LACP) sous Linux
$ cat /etc/network/interfaces
auto bond0
iface bond0 inet static
    address 192.168.1.10
    netmask 255.255.255.0
    bond-slaves eth0 eth1
    bond-mode 802.3ad       # LACP — agrégation active/active
    bond-miimon 100
// adresse MAC
Une adresse MAC est composée de 6 octets hexadécimaux : les 3 premiers identifient le fabricant (OUI — Organizationally Unique Identifier), les 3 derniers sont l'identifiant unique de l'appareil. Exemple : a4:c3:f0:1b:2d:3e → a4:c3:f0 = Intel Corporation.

08 VLANs — Réseaux locaux virtuels

Un VLAN (Virtual LAN) permet de segmenter logiquement un réseau physique en plusieurs réseaux isolés, sans avoir besoin de câbles séparés. Les machines d'un même VLAN communiquent librement ; celles de VLANs différents doivent passer par un routeur ou un switch L3.

Les VLANs sont définis par la norme 802.1Q : un tag de 4 octets est ajouté à la trame Ethernet, contenant l'identifiant du VLAN (VID, de 1 à 4094).

# Configuration VLANs sur un switch Cisco IOS

# Créer les VLANs
Switch(config)# vlan 10
Switch(config-vlan)# name BUREAUX
Switch(config)# vlan 20
Switch(config-vlan)# name SERVEURS
Switch(config)# vlan 99
Switch(config-vlan)# name GESTION

# Assigner un port en mode access (un seul VLAN)
Switch(config)# interface GigabitEthernet0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10

# Configurer un port trunk (plusieurs VLANs, vers un autre switch ou routeur)
Switch(config)# interface GigabitEthernet0/24
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,99

Un switch de couche 3 combine les fonctions d'un switch L2 et d'un routeur. Il peut faire du routage inter-VLAN à très haute vitesse (en hardware), ce qui réduit le besoin de routeurs dédiés dans le cœur d'un réseau d'entreprise.
